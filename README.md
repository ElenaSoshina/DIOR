# DIOR

Статический многостраничный сайт-визитка (HTML/CSS/JavaScript) с кастомными анимациями и интерактивными секциями: главная, мастерская, обучение, партнёры.

## Быстрый старт

### Вариант 1: открыть напрямую

1. Клонировать или скачать проект.
2. Открыть файл `index.html` в браузере.

Подходит для быстрой проверки вёрстки, но для корректной проверки относительных путей и поведения в браузере лучше использовать локальный HTTP-сервер (вариант 2).

### Вариант 2 (рекомендуется): локальный сервер

Из корня проекта:

```bash
python3 -m http.server 8080
```

Затем открыть:

- `http://localhost:8080/index.html`

Остановка сервера: `Ctrl + C`.

## Технологический стек

- **HTML5** — структура страниц.
- **CSS3** — стилизация, адаптивность, анимации (`@keyframes`, `transform`, `media queries`).
- **Vanilla JavaScript (ES6)** — интерактивность, карусели, табы, модальные окна, навигация.
- **Google Fonts** — подключение шрифтов через `@import` в CSS.
- **Без backend** — проект полностью фронтенд-статический.
- **Без сборщика и пакетного менеджера** — нет `package.json`, `webpack`, `vite`, `npm scripts`.

## Архитектура проекта

Проект организован как набор отдельных HTML-страниц, каждая со своим CSS и JS (кроме главной, где используются `main.css` и `script_main.js`).

### Основные файлы

- `index.html` — главная страница с интерактивными «ромбами» (переходы между разделами).
- `main.css` — стили главной страницы.
- `script_main.js` — логика кликов по ромбам.

- `education.html` — страница «Обучение» с 3D-каруселью.
- `education.module.css` — стили страницы обучения.
- `script3.js` — логика ротации карусели, индикаторов и мобильного меню.

- `workshop.html` — страница «Мастерская» (галерея + форма заявки + контакты).
- `workshop.module.css` — стили страницы мастерской.
- `script_workputshop.js` — слайдер, стрелки, модальное окно просмотра фото.

- `partners.html` — страница «Партнёры» с вкладками и большим текстовым контентом.
- `partners.module.css` — стили страницы партнёров.
- `script_partners.js` — переключение вкладок, «Подробнее/Свернуть».

- `images/` — статические изображения и иконки.
- `README.md` — техническая документация и статус текущих ограничений.

### Рекомендуемая точка входа

- Всегда запускать/тестировать с `index.html`.

## Навигация и поведение страниц

## 1) Главная (`index.html`)

- Шапка с desktop/mobile меню.
- Основной SVG-фон и 4 кликабельные области (`#diamond1..#diamond4`).
- Переходы задаются в `script_main.js`:
  - `diamond3` → `education.html`
  - другие ромбы сейчас ведут на `page1.html`/`page2.html` (файлы отсутствуют, см. раздел «Текущее состояние и ограничения»).

## 2) Обучение (`education.html`)

- 3D-карусель из 15 элементов.
- Индикаторы с активным состоянием.
- Скрипт `script3.js`:
  - управляет паузами/скоростью вращения;
  - использует разные интервалы для desktop и mobile;
  - позволяет кликать по индикаторам;
  - переключает мобильное меню.

## 3) Мастерская (`workshop.html`)

- Динамически создаваемый слайдер изображений из массива `photos` (`script_workputshop.js`).
- Левая/правая навигация по слайдам.
- Модальное окно для просмотра изображения в увеличенном виде.
- Контактный блок + форма заявки (`action="#"`, без отправки на сервер).

## 4) Партнёры (`partners.html`)

- Переключаемые вкладки по компаниям.
- Контент вкладок частично ограничен по высоте.
- Кнопка «Подробнее» разворачивает полный контент вкладки.

## 5) Отзывы / благодарственные письма

- Отдельная страница/секция в текущей версии проекта **не реализована**.
- Пункты меню «Отзывы» пока ведут на `#` и не открывают контент.

## Текущее состояние и ограничения

Ниже важные пункты, которые нужно знать перед доработкой проекта:

1. **Часть изображений отсутствует в репозитории**
   - В коде используются, но не найдены в `images/`:
     - `images/main_bg.png`
     - `images/map.jpeg`
     - `images/dot_active.png`
     - `images/image1.png` ... `images/image7.png`
     - `images/workshop/1.jpeg` ... `images/workshop/5.jpeg`
   - Без них часть страниц будет отображаться с пустыми блоками/битым фоном.

2. **Есть ссылки на несуществующие страницы**
   - В `script_main.js` используются `page1.html` и `page2.html`, но этих файлов нет в проекте.

3. **Мобильное меню на главной**
   - В `index.html` используется `onclick="toggleMenu()"`, но в `script_main.js` функция `toggleMenu` отсутствует.

4. **Форма обратной связи статическая**
   - В `workshop.html` форма не интегрирована с API/почтой (`action="#"`).

5. **Секция "Отзывы / благодарственные письма" отсутствует**
   - В навигации пункт присутствует, но отдельная страница с каруселью не реализована.

6. **Важная деталь по стрелкам в "Мастерской"**
   - Был конфликт между общим классом `back-arrow` и стрелками слайдера.
   - Текущее состояние: конфликт устранён стилем внутри `workshop.module.css` (`.arrows .back-arrow`), правая стрелка развернута через `rotate(180deg)`.

## Как передавать проект следующему разработчику

Рекомендуемый минимальный onboarding-чеклист:

1. Поднять локальный сервер и проверить все 4 страницы.
2. Восстановить отсутствующие ассеты (`images/...`) от дизайнера/предыдущей команды.
3. Привести в порядок маршрутизацию (заменить/добавить `page1.html`, `page2.html` или обновить ссылки).
4. Унифицировать меню (desktop/mobile) на всех страницах.
5. Реализовать страницу/секцию «Отзывы / благодарственные письма» (планировалась карусель).
6. Решить, нужна ли сборка (Vite/Webpack) или проект остаётся полностью статическим.

## Локальная разработка

### Изменение контента

- Текст и структура разделов — прямо в соответствующих `*.html`.
- Стили конкретной страницы — в её `*.module.css`.
- Логика страницы — в связанном `*.js`.

### Добавление новых изображений

- Класть файлы в `images/` (или подпапки, например `images/workshop/`).
- Использовать относительные пути по текущей структуре.
- Проверять регистр имён файлов (в некоторых окружениях путь чувствителен к регистру).

### Проверка перед коммитом

- Проверить консоль браузера на JS-ошибки.
- Пройти вручную сценарии:
  - переходы с главной;
  - работа каруселей;
  - открытие/закрытие модалки;
  - переключение табов и «Подробнее».
